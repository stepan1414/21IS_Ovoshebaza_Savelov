## 1. Описание проекта
Данный проект представляет собой прототип Овощной базы для удобства нахождения и поиска продуктов, учёта продаж, регистрации пользователей.  
Система позволяет структурировать хранение данных и быстро получать аналитическую информацию через SQL-запросы.

##  2. Техническое задание (ТЗ) вкратце
*   **Цель:** Цифровизация учета овощей: от ручных журналов к реляционной базе данных на SQLite.
*   **Основные функции:**
   1. Регистрация поставщиков и поступлений товара.
   2. Учет проданных единиц продукции.
   3. Контроль текущих остатков на складе.
   4. Формирование отчетов по выручке за период.
*   **Требования к БД:** Минимум 4 связанные таблицы, соблюдение целостности данных.

##  3. Структура базы данных (ER-схема)
Проект базируется на следующих сущностях:
1.  **Продукция]** — справочник овощей (название, сорт, цена за кг).
2.  **[Сорта]** — справочник сортов овощей с привязкой к продукту (например, Картофель → сорт "Гала", "Колобок", "Невский").
3.  **[Склады]** — места хранения продукции (адрес, тип хранения: охлаждаемый/сухой).
4.  **[Закупки]** - фиксирует закупку товара (связь: сорт, поставщик, склад, количество, цена закупки, дата).

*Скриншот ER-диаграммы находится в папке `/diagrams`.*

##  4. Реализованные SQL-запросы
В рамках проекта подготовлены сценарии для:
1. Подсчета текущих остатков: Расчет разницы между суммой поступлений и продаж по каждому овощу.
2. Выборки с JOIN: Вывод чеков продаж с названиями продуктов и их ценой вместо ID.
3. Фильтрации: Поиск поставщиков, которые поставляли товар в заданный период.
4. Аналитики: Топ-3 самых продаваемых овощей за месяц.

##  Структура репозитория
/data — файл базы данных farm_fresh.db.
/sql — скрипты создания таблиц (schema.sql) и тестовые данные (seeds.sql).
/diagrams — графическая схема базы данных (er_diagram.png).
queries.sql — файл с примерами сложных запросов (отчеты).

##  Как запустить (Инструкция)
1. Установите DB Browser for SQLite или используйте консоль sqlite3.
2. Склонируйте репозиторий: git clone [ссылка на ваш репозиторий].
3. Откройте файл /data/farm_fresh.db или выполните последовательно скрипты из /sql/ для создания БД с нуля.
4. Запустите запросы из файла queries.sql для проверки работоспособности.
